Есть система, обрабатывающая фин. транзакции.
Необходимо написать микросервис, определяющий правило, по которому транзакция должна быть обработана

Каждая транзакция представляет собой набор параметров
- тип транзакции(код типа из справочника);
- направление транзакции (оригинал /отмена)
- источник транзакции (код системы из справочника)
- сумма транзакции
- валюта транзакции
- дата транзакции

каждое правило состоит из условий на параметры транзакции:
тип(равно)
направление(равно)
Источник(равно)
Сумма(больше/меньше/равно)
И действие, которое необходимо выполнить (Провести/отклонить)
Код типа баланса

Все правила имеют срок действия(дата начала, дата окончания)

При выборе правила по транзакции, используются следующие критерии:
правило действительно на момент проведения транзакции.
транзакция удовлетворяет всем условиям на правиле(если какое либо из условий не задано, то  считается, что транзакция ему удовлетворяет)
Если транзакция удовлетворяет нескольким правилам, то выбирается то правило, у которого задано больше условий.
Если кол-во условий одинаково, то выбирается то правило, которое было создано позднее.


Ответ сервиса должен содержать действие, заданное правило и уникальный идентификатор правила.

Нефункциональные требования:
- правила храняться в СУБД
- UI для управления правилами не нужен.
